# 对话界面人物形象设计

## 目标

将调查页中间的 `NpcDialoguePanel` 从普通记录列表升级为真实对话界面。界面采用“档案证言板”方向：每条消息以聊天气泡呈现，气泡两侧显示说话者形象。NPC 在左侧，玩家“我”在右侧；当前地点没有可对话 NPC 时，仅显示玩家本人和空状态提示。

本设计只改变前端表现和前端角色形象映射，不改变案件真相、NPC 知识边界、线索收集规则、AI prompt 权限或后端数据结构。

## 视觉方向

采用用户确认的 B 方案“档案证言板”。整体仍延续现有暗绿色、旧纸色、蜡黄高光和怪谈调查局风格。

对话区由三层组成：

1. 顶部证言栏：保留 NPC 选择器，显示当前对话对象的姓名、身份和状态。
2. 中部证言记录：每条对话是一组“头像/立绘卡 + 气泡”。NPC 消息左对齐，玩家消息右对齐。
3. 底部提问区：保留快捷问题栏和自由输入框。

气泡需要像档案记录而不是现代 IM：边框克制、纸纹暗底、轻微阴影，玩家气泡使用蜡黄色强调，NPC 气泡使用旧纸灰绿底色。

## 角色形象

第一版不引入外部图片资源，使用前端 CSS 绘制的角色形象组件，避免新增图片生成和素材管理成本。每个 NPC 通过 `code` 映射到独立形象配置。

角色形象包含：

- 头像/半身剪影容器。
- 中文姓名首字或称号标记。
- 主色、辅色、边框色。
- 小型身份符号，例如钟、钥匙、船票、雾灯、账本等，用文字或 CSS 形状表达。
- “我”的固定调查员形象，出现在玩家气泡右侧。

不同案件的 NPC 形象应该有区分度：

- 钟声案件偏钟楼、矿难、红手套、旧教堂元素。
- 旅馆案件偏钥匙、账本、镜面、前台铃元素。
- 雾港案件偏船票、雾灯、码头、船员帽元素。

如果遇到未配置的 NPC，使用默认“镇民证言人”形象，不让界面空白。

## 交互规则

选择 NPC 后，顶部栏和后续 NPC 气泡使用该 NPC 的形象。玩家发送的问题永远使用“我”的形象。

当前地点有 NPC 时：

- 显示 NPC 下拉选择。
- 空对话时显示一条带当前 NPC 形象的引导空状态。
- 发送问题后，玩家气泡在右，NPC 回复在左。

当前地点没有 NPC 时：

- 不显示对方 NPC 形象。
- 中部只显示玩家调查员形象和“此处暂无可询问对象”的空状态。
- 输入区应禁用或给出明确提示，避免玩家误以为卡住。

## 组件边界

新增 `CharacterPortrait` 组件，职责是根据角色配置绘制形象，不参与对话业务逻辑。

`NpcDialoguePanel` 负责：

- 计算当前 NPC。
- 将消息渲染为左右气泡行。
- 在 NPC 不存在时展示玩家空状态。
- 保持原有 `selectNpc` 和 `ask` 事件接口。

`InvestigationView` 尽量不扩展职责，只继续传入 `npcs`、`selectedNpcId`、`messages` 和 `loading`。

## 测试策略

前端使用 Vitest 和 Vue Test Utils 增加组件测试：

- NPC 消息渲染在左侧，并显示对应 NPC 形象。
- 玩家消息渲染在右侧，并显示“我”的形象。
- 每个已知 NPC code 能得到独立形象配置，未知 code 有默认形象。
- 没有 NPC 时只显示玩家空状态，且提问按钮不可用。

实现后运行：

```bash
npm run test
npm run build
```

## 非目标

本次不新增后端字段，不改 SQL，不接入图片上传，不让 AI 生成或改写角色形象，不改变对话消息存储方式，也不改变案件固定真相。
